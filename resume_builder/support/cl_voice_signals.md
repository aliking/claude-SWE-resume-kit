## Format

Each entry block:

```
## [Company] [context] ([date]) — [session path]
- "[verbatim or near-verbatim user phrase]" — [signal note]
- Tone shift: [before] → [after]
```

Scope rules:
- Store wording signals only (verbatim or near-verbatim user language).
- Do not store structural mandates (e.g., "always remove the breadth paragraph").
- Keep newest additions at the top when appending new entries.
- Current-session motivation inputs still have highest priority.
